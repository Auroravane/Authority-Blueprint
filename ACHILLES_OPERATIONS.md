# 🛡️ Project ACHILLES — Operational Manual

This repository contains the **ACHILLES Market Intelligence Engine**. If this repository is ever cloned to a new environment or the local files are lost, follow this guide to restore the autonomous workflow.

## 🏗️ Core Architecture
The system is designed to be "Zero-Infrastructure" and "Zero-Cost," running entirely on GitHub Actions and NVIDIA NIM.

1.  **System Brain**: `SystemPrompt.md` (Contains 1,200+ lines of strategic logic).
2.  **Engine**: `scripts/report_engine.py` (The Python bridge between AI and Git).
3.  **Scheduler**: `.github/workflows/daily_report.yml` (Runs daily at 3 AM JST).

## 🚀 How to Restore from Scratch

### 1. Requirements
*   A GitHub Repository.
*   An NVIDIA NIM API Key (Mistral-Nemotron model).
*   The three files mentioned above.

### 2. GitHub Secrets Configuration
You **must** set the following secret in `Settings > Secrets and variables > Actions`:
*   `NVIDIA_API_KEY`: Your private key from NVIDIA.

### 3. Execution Logic
*   **Input**: The engine reads `SystemPrompt.md` and asks the AI for a "Format A: Weekly Signal Report."
*   **Output**: The bot generates Markdown files and commits them to the `reports/` folder.
*   **Downstream Access**: Any other workflow can access the latest intelligence by reading `reports/latest.md`.

## 🛠️ Manual Maintenance
*   **To Test**: Go to the GitHub "Actions" tab and manually trigger "Daily ACHILLES Report".
*   **To Update Strategy**: Edit `SystemPrompt.md`. The bot will immediately adopt the new strategy on its next run.
*   **Timezone**: The bot is currently set to **3:00 AM Japan Standard Time** (18:00 UTC).
